O que é o Squid?
É um proxy cache utilizado em provedores de
Internet e rede de empresas.
Mas o Squid vai além do cache.
Também possui:
  • Controle de acessos;
  • Autenticação;
  • Controle de banda;
  • Logs de acesso (é possível gerar relatórios a
    nível de usuário com ferramentas adicionais).
O que se ganha com o
Squid?
  • Usar menos tráfego de Internet;
  • Reduzir o tempo de carregamento das páginas;
  • Proteger os hosts da rede interna;
  • Coletar estatísticas de acesso;
  • Prevenir que usuários acessem conteúdo
    indevido ou não permitido;
  • Certificar-se de que apenas usuários
    autorizados acessem à Internet;
O que se ganha com o
Squid?
  • Melhorar a privacidade do usuário filtrando
    requisições web de conteúdo confidencial;
  • Reduzir a carga do seu próprio servidor web
    (proxy reverso);
  • É possível limitar a velocidade de acesso a sites
    ou de computadores (para acessos pelo proxy).
  • Permite utilizar ferramentas extras, como
    geradores de relatórios (Sarg) ou
    redirecionadores (SquidGuard e ufdbGuard).
Quais as limitações?
  • Um proxy não substitui um firewall (mas
    podem ser utilizados em conjunto).
  • Apesar de ser gratuito (e Open Source), o
    custo de manutenção pode ser alto.
  • O proxy transparente não pode ser utilizado
    com HTTPS, necessitando configuração
    manual.
Se o Squid é tão
bom, quais são
suas "possíveis"
ameaças?
Cache como vilão
  • Incompatibilidade com sites (Ex: Conectividade
    Social Caixa, Ipiranga - só burlando RFC).
  • Menor quantidade de sites estáticos.
  • O custo do cache pode ser alto, pois exige HDs
    rápidos e grande armazenamento.
  • Casos em que seria vantajoso o uso de cache
    (como Windows Update, YouTube) nem
    sempre são permitidos.
  • Links grandes para pequenas empresas (sem
    necessidade de cache). Em alguns casos é mais
    barato usar um Mikrotik.
HTTPS
  • Os sites estão migrando para o HTTPS.
  • Forçando a implementação, o Google Chrome
    vai exibir um alerta de site inseguro para todas
    as páginas que não forem HTTPS e solicitam
    alguma informação, como usuário e senha, por
    exemplo (ref.: BR-Linux -
    http://bit.ly/brlinuxssl).
Outros proxies
  • Existem outras ferramentas de proxy, não tão
    abrangentes quanto o Squid, mas focadas no
    que propõe.
  • Tinyproxy: bom para hardware com pouca
    memória e processamento. Ideal para pequenas
    empresas (free).
  • Polipo: proxy simples e rápido para fazer cache
    e filtrar sites (free).
  • Ferramentas de cache para provedores (ou
    grandes empresas): RaptorCache, Taghos etc.
    São normalmente pagas.
Firewalls "Appliance"
  • Ferramentas poderosas (mas sem tanta
    liberdade como no squid).
  • Podem até facilitar o gerenciamento.
  • Ex.: Palo Alto, SonicWALL,
    FortiNet/FortiGate.
E o Squid?
  • Existem empresas que o cache ainda é um bom
    investimento. O controle de banda e bloqueios
    que o Squid oferece também ajudam no
    aproveitamento do link de Internet.
  • Apesar da praticidade de appliances, nem todas
    as empresas possuem pessoas capacitadas,
    necessitando terceirizar o gerenciamento da TI.
  • Possibilita autenticação com o AD.
  • Ao usar ferramentas extras, é possível otimizar
    o Squid: Thunder Cache (otimização de cache),
    Webmin (gerenciamento), squidGuard,
    ufdbGuard, DansGuardian.
Deve-se analisar
o melhor para
cada cenário.
